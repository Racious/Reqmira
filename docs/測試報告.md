## 四、手動驗收清單

> 於 `npm start` 視窗、開啟 `workspace/` 後依序驗收。

1. **送出/環境**：demo 環境 → Demo·httpbin GET → 送出 → 200、JSON 樹。
2. **動態變數**：URL 加 `?id={{$uuid}}&t={{$timestamp}}` → 送出 → 回應 args 見即時值。
3. **Auth 分頁**：選 Bearer 填 `{{token}}` → 送出 → httpbin headers 見 `Authorization: Bearer …`。
4. **Response 過濾**：Pretty 過濾框輸入 `$.headers` 只看該段。
5. **Codegen / Diff / History**：產 TS/Java/MD；設基準→再送→Diff 對照；History 列藍框（檢視中）/金鈕（基準）。
6. **Flow**：Flows → Demo 兩步流程 → ▶ 執行 → 全部通過。
7. **樹管理**：右鍵資料夾新增/重新命名/刪除；**拖**請求到資料夾或最外層空白區。
8. **佈局**：頂列 `⇆`/`⇅` 切換 Request/Response 上下↔左右；Inspector 可收合、欄寬可拖。
9. **複製反饋**：點「值/⧉」→ 底部綠勾 toast。

## 五、開發中遇到的問題與修復紀錄

| # | 問題 | 根因 | 修復 |
|---|---|---|---|
| 1 | 鷹架名 `_scaffold` | create-tauri-app 暫存名 | 全面改為 Reqmira |
| 2 | `npm start` port 1420 占用 | 預覽面板殘留 dev server | 查殺占用程序後重啟，後續一次到位 |
| 3 | 建構 TS2345 | `{ type:"mixed" }` 未窄化 | 明確標註 `SchemaNode[]` |
| 4 | `diff_json` never-used 警告 | 註冊前的過渡建置 | 註冊後正常 |
| 5 | 查回來卡頓 | 重複 parse / 展開過深 / 大陣列 / debug build / client 重建 | Schema 共用解析、收斂展開、大陣列截斷、dev opt-level、client 重用 |
| 6 | `toTypeScript` 陣列根遺失巢狀 interface | 提前 return 丟棄 defs | 修正並加回歸測試 |
| 7 | `parse_method("WAT!!")` 測試失敗 | 誤判——`!` 為合法 token，HTTP 允許自訂方法 | 修正測試假設 |
| 8 | **拖拉顯示「禁止」游標** | Tauri 預設 `dragDropEnabled:true` 攔截 webview HTML5 DnD | tauri.conf 視窗加 `dragDropEnabled:false` 並重編 |
| 9 | 拖拉無法移到最外層 | 根層無放置區 | Collections 區加根層 drop zone，資料夾 drop 阻冒泡 |
| 10 | Diff「不比對」 | 比同一筆 / httpbin 相同請求回應一致 | 加「同一筆/內容相同」提示、點分頁直接觸發、不再強制切走分頁 |
| 11 | 基準無法區別當前/基準 | 回應區設基準未關聯 History；無「檢視中」標示 | 設基準關聯 History id（該列高亮）；「基準」鈕改開關高亮；檢視中列加藍框 |
| 12 | 前端改動「看不到」 | 多為 HMR 未整頁重載 / 視窗未重啟 | 說明 F5 重載；Rust 變更靠 watcher 重編；必要時重啟 dev |
| 13 | **網路 e2e 偶發失敗** | httpbin 外部瞬斷（回非 200）；**非 Reqmira 回歸**（send 本身成功） | 重跑即過；建議以 wiremock 取代外網，使 e2e 不受外部影響 |

## 六、已知測試缺口與建議

1. **網路 e2e 去外部化**：以 `wiremock` mock server 取代 httpbin，消除偶發失敗（#13）。
2. **Auth 組裝邏輯**：`buildAuth`（前端 session 內）目前靠手動驗收；可抽出為純函式以 Vitest 覆蓋。
3. **元件測試**：`KeyValueEditor`/`SchemaTree` 可用 Vitest + `@vue/test-utils` 補。
4. **GUI 端到端**：Tauri 互動無法在瀏覽器自動測，現以手動清單補足。

## 七、結論

- **核心邏輯（發送/解析/diff/型別/產生/路徑/auth 持久化）皆有自動化測試並全數通過**：前端 34、後端 24（+1 外部 e2e 重跑通過）。
- 過程中發現並修復一個真實 bug（陣列根 codegen）、釐清一處測試誤判、解決拖拉與 Diff/基準等多項 UX 問題。
- 唯一非綠是**外部依賴**（httpbin 瞬斷）導致的 e2e 偶發失敗，重跑即過，已列建議去外部化。
- 專案建構、型別檢查、前後端測試全綠，處於健康可持續開發狀態。
